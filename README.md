# Загрузка Docker образов
Этот инструмент предназначен для загрузки Docker контейнеров в случае отсутствия доступа к репозиторию Dockerю

Основная идея — использовать GitHub Codespaces или аналогичный сервис для загрузки образов. Скрипт скачивает образ внутри облачной инфраструктуры GitHub, сжимает его "на лету" и создает готовый архив. После этого можно скачать готовый файл на свою локальную машину через обычный браузер (HTTPS) или по SSH для использования на локальной машине.

## Системные требования к ПО
* Скрипт создавался для Debian based дистрибутивов:
  * Debian 10 Buster и новее.
  * Ubuntu 18.04 и новее.
* Для скачивания образов используется утилита skopeo.
* **Для сжатия и распаковки архива с образом используется утилита `zstd`. Проверьте ее наличие на локальной машине для распаковки архива.**

## Требования к машине Github Codespaces
Скрипт рассчитан на работу с минимальной конфигурацией Github Codespaces, а именно:
* Процессор amd64 от 2 ядер.
* RAM от 4 gb.
* Storage от 32gb.
* GPU - не требуется.

## Инструкция
Предполагается что у вас развернут настроен Github Codespace, вы подключены к нему через браузер или по SSH.

### Шаг 1 Установка зависимостей
Скопируйте и вставьте в терминал команду для установки зависимостей в Github Codespace.
```bash
sudo apt update --allow-insecure-repositories && sudo apt install skopeo -y
```
*На момент тестирования скрипта были проблемы с GPG ключами в репозитории Ubuntu.*

### Шаг 2 Скачивание образа
1. Скопируйте команду ниже в удобное место для редактирования (`\` в начале тоже, это часть команды).
2. Отредактируйте переменные в блоке **Настройки**:
 * SOURCE_IMAGE: Полный путь к образу (например, nginx:latest или ghcr.io/...).
 * OUTPUT_FILE: Имя будущего файла (без расширения).
 * COMPRESS: Степень сжатия от 1 до 22, оптимальными являются от 4 до 19, оптимальная 15, не рекомендуется изменять без необходимости.
3. Скопируйте отредактированную команду и вставьте в терминал Github Codespace.
4. Запустите команду нажав **Enter**.
```bash
\
####################################Настройки#####################################
SOURCE_IMAGE=""
OUTPUT_FILE=""
COMPRESS="15"
##################################################################################
FULL_PATH="${OUTPUT_FILE}.tar.zst"
set -o pipefail

echo "Downloading and compressiing..."
skopeo copy --quiet \
  docker://"$SOURCE_IMAGE" \
  docker-archive:/dev/stdout:"$SOURCE_IMAGE" | \
  zstd -"$COMPRESS" -T0 --long=27 -v > "$FULL_PATH"

echo "Download checking..."
if [ $? -ne 0 ]; then
    echo "DOWNLOAD OR COMPRESS ERROR!"
    exit 1
else
    echo "Download complite."
fi


echo "Testing..."
if zstd -t "$FULL_PATH" && \

echo "Reading..." && \
zstd -dc "$FULL_PATH" | tar -tf - > /dev/null; then
	du -sh "$FULL_PATH"
	echo "OK"
else
    echo "ERROR!"
	exit 1
fi
##################################################################################
```

### Шаг 3 Скачивание образа
После скачивания образа скрипт автоматически его протестирует, если вы видите сообщение `OK` и успешное завершение работы скрипта скачайте Docker образ на локальную машину.

В Github Codespace нажмите правой кнопкой мыши на созданный архив с расширением `.tar.zst` и нажмите **Загрузить**, укажите папку для загрузки и дайте разрашение на скачивание файла. При использовании ssh передайте файл на вашу локальную машино по SFTP.

### Шаг 4 Распаковка и установка скаченного образа
Переместите архив с образом на локальную машину с установленным Docker и выполните коману.
```bash
zstd -dc путь_до_архива_или_его_имя.tar.zst | docker load
```

После установки образа проверьте корректность командой `docker images`, образ должен отображаться в списке.


## Технические детали скрипта
| Параметр | Функция |
|---|---|
| skopeo copy | Скачивает образ напрямую из реестра без использования локального Docker-демона. |
| docker-archive:/dev/stdout | Направляет поток данных образа прямо в "трубу" (pipe), минуя запись на диск. |
| zstd -15 | Высокий уровень сжатия (оптимально для баланса скорости и размера). |
| --long=27 | Увеличивает окно поиска дубликатов до 128 МБ (очень эффективно для слоев Docker). |
| set -o pipefail | Гарантирует, что скрипт остановится, если скачивание прервется. |

## Важные замечания
 * Скрипт оптимизирован под 4-8 ГБ RAM. Если Codespaces начинает тормозить, снизьте уровень сжатия `COMPRESS` до 10.
 * Помните, что если вы скачиваете образ в Codespaces (обычно это amd64), он может не подойти для ARM, если образ не является multi-arch.
 * Для распаковки требуется больше оперативной памяти чем для архивации, учитывайте это при использовании и настройке `COMPRESS`, рекомендуется не мнеее 8gb.
